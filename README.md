# 🪿 reel-vision

The frontend for **[GooseGram](https://github.com/jenniedov/goosegram)** — an
AI-curated Instagram Reels feed that filters out anxiety bait and surfaces
what actually feels good.

This is a Vite + TanStack Start + Tailwind v4 + shadcn/ui app that reads
scored reels from a Supabase project and renders them in an
Instagram-style vertical feed (gradient avatar ring, verified checkmark,
Follow button, action rail with heart/comment/repost/DM, comment sheet,
bottom nav).

## Routes

| URL | What |
|---|---|
| `/onboarding` | First-visit onboarding flow (cosmetic; sets `localStorage.hg_onboarded`) |
| `/` | The curated reels feed (`approved_for_demo=true AND blocked=false ORDER BY display_order`) |
| `/?nodemo=true` | The live agent feed (`approved_for_demo=false`), shows whatever the agent has scraped+kept |
| `/filteredout` | The reels the agent BLOCKED, with the `block_reason` (anxiety score, etc.) |

## Quick start

```bash
git clone https://github.com/jenniedov/reel-vision.git
cd reel-vision
bun install

# Wire your Supabase project
echo 'EXTERNAL_SUPABASE_ANON_KEY="sb_secret_..."' > .env.local
# Edit src/lib/reels.functions.ts EXTERNAL_URL to your project ref

bun run dev   # → http://localhost:8080
```

Note `EXTERNAL_SUPABASE_ANON_KEY` is read **server-side only** by the
TanStack Start server function at `src/lib/reels.functions.ts`. It never
ships in the client bundle. Use a service role key for now (RLS lets the
table be public-readable).

For Lovable hosting: set `EXTERNAL_SUPABASE_ANON_KEY` in
**Project Settings → Environment Variables** instead of `.env.local`.

## Where the data comes from

Run the [GooseGram agent](https://github.com/jenniedov/goosegram) to populate
the `reels` table — the agent handles the Apify scrape, Claude scoring,
video frame extraction, and Supabase writes. This frontend is just a reader.

## Stack

- **Vite 7** + **TanStack Start** + **TanStack Router** + **TanStack Query**
- **Tailwind v4** + **shadcn/ui** components
- **React 19**
- **lucide-react** for icons
- Generated by [Lovable](https://lovable.dev), then customized.
